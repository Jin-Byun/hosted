what is SSL

  • provides encryption, security layer for HTTPS (purchase SSL from provider to be able to host HTTPS page)
  • with HTTPS, browser and server establish encrypted conn thru SSL/TLS handshake
  • to get SSL certificate, you must provide address, credit card number, etc. they can be tracked
  • Also, it allows point to point connection, meaning that there wouldn't be anyone else that can access your data.

OAuth and third party auth

  • using oAuth, no need for authentication implementation.
  • oAuth does one more thing, which is granting permisson to share information stored in other app/website
  • std, secure, display google login, etc.

2 layers

register website to google (onboarding)

  1. https://console.developers.google.com
  2. give login page, user landing page,
  3. upon approval, google will create client key(pub) and secret key(secret for client App)
  4. along with the keys, the snippet for google login is given.

user side

  1. on login, redirect to OAuth 2 endpoint and include client App id/key
  2. use the client id/key to display the form
  3. upon user approval, google generates bearer token and redirect to landing page.
  4. the bearer token is then stored in the database
  5. refresh token exists, and if bearer token exires, use refresh token to get new bearer and refresh token.
